**Билет 3. Диаграмма «Сущность-связь»**

### Назначение
Диаграмма «Сущность–связь» (ERD — Entity-Relationship Diagram) визуализирует структуру данных на концептуальном уровне: сущности, их атрибуты и связи между ними.

### Основные нотации
1. **Чен (Chen notation)** — оригинальная нотация:
   - Сущности — прямоугольники.
   - Атрибуты — овалы, связанные с сущностями.
   - Связи — ромбы, соединяющие сущности.
   - Кратность указывается текстом рядом со связью.

2. **Crow’s Foot (Нога вороны)** — популярна в промышленности:
   - Сущности — прямоугольники с колонками (атрибутами).
   - Связи — линии с символами на концах:
     - Одинарная черта — «один»,
     - Тройная — «много»,
     - Круг — необязательность участия.

3. **UML-подобная нотация**:
   - Использует классы UML для представления сущностей.
   - Связи отображаются как ассоциации с кратностью (например, `1`, `0..*`).

4. **Barker’s notation** — используется в Oracle Designer:
   - Похожа на Crow’s Foot, но с дополнительными соглашениями по ключам и обязательности.

В PostgreSQL такие диаграммы помогают проектировать таблицы до написания DDL:
```sql
-- Пример: сущность "Автор" связана с "Книга" (1:N)
CREATE TABLE Автор (id SERIAL PRIMARY KEY, имя TEXT);
CREATE TABLE Книга (id SERIAL PRIMARY KEY, название TEXT, автор_id INT REFERENCES Автор(id));
```
