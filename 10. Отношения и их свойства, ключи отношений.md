**Билет 10. Отношения и их свойства, ключи отношений**

### Свойства отношения (реляционной таблицы)
В реляционной модели **отношение** — это математическое множество кортежей (строк), обладающее следующими свойствами:
- **Уникальность строк**: дубликаты запрещены.
- **Неупорядоченность строк**: порядок не имеет значения.
- **Атомарность атрибутов**: каждый атрибут (столбец) содержит скалярное значение (1NF).
- **Именованные столбцы**: каждый атрибут имеет уникальное имя в рамках отношения.
- **Однородность типов**: все значения в столбце одного типа.

### Типы ключей
- **Суперключ** — набор атрибутов, однозначно идентифицирующий кортеж.
- **Кандидатный ключ** — минимальный суперключ (удаление любого атрибута нарушает уникальность).
- **Первичный ключ (Primary Key)** — один из кандидатных ключей, выбранный для идентификации записей.  
  В PostgreSQL:  
  ```sql
  CREATE TABLE users (id SERIAL PRIMARY KEY, email TEXT UNIQUE);
  ```
- **Альтернативный ключ** — любой кандидатный ключ, не выбранный как первичный (например, `email` выше).
- **Внешний ключ (Foreign Key)** — атрибут, ссылающийся на первичный ключ другой таблицы, обеспечивая ссылочную целостность:  
  ```sql
  CREATE TABLE orders (
    id SERIAL PRIMARY KEY,
    user_id INT REFERENCES users(id)
  );
  ```

Ключи лежат в основе нормализации и поддержания целостности данных в реляционных СУБД.
