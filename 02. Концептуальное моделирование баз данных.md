**Билет 2. Концептуальное моделирование баз данных**

### Сущности
- **Сильная сущность** — существует независимо (имеет собственный первичный ключ).  
  Пример: `Покупатель(id, имя)`.
- **Слабая сущность** — зависит от сильной и не имеет уникального ключа без неё.  
  Пример: `Заказ(номер, дата)` зависит от `Покупатель`.

### Атрибуты и их типы
- **Простые** — неделимые (`возраст`).
- **Составные** — состоят из податрибутов (`адрес = {улица, город}`).
- **Многоценные** — хранят несколько значений (`телефоны`).
- **Производные** — вычисляются из других (`возраст` ← `дата_рождения`).

### Связи (Relationships)
- **Степень связи** — количество участвующих сущностей:  
  - бинарная (2), тернарная (3) и т.д.
- **Кратность (кардинальность)** — ограничения количества связей:
  - «один к одному» (1:1),
  - «один ко многим» (1:N),
  - «многие ко многим» (M:N).

В реляционной модели связь M:N требует промежуточной таблицы:
```sql
CREATE TABLE Покупатель_Товар (
  покупатель_id INT REFERENCES Покупатель(id),
  товар_id      INT REFERENCES Товар(id),
  PRIMARY KEY (покупатель_id, товар_id)
);
```
