**Билет 19. Операторы манипулирования данными INSERT и UPDATE в языке SQL**

### `INSERT` — добавление данных
**Синтаксис**:  
```sql
INSERT INTO table_name [(col1, col2, ...)]
VALUES (val1, val2, ...)
[RETURNING * | columns];  -- PostgreSQL-специфика
```

**Примеры**:  
— Вставка одной записи:  
```sql
INSERT INTO users (name, email) VALUES ('Алиса', 'alice@example.com');
```
— Массовая вставка:  
```sql
INSERT INTO products (name, price) VALUES 
  ('Товар A', 100.0),
  ('Товар B', 200.5);
```
— Вставка с возвратом ID (PostgreSQL):  
```sql
INSERT INTO logs (event) VALUES ('login') RETURNING id;
```

### `UPDATE` — изменение данных
**Синтаксис**:  
```sql
UPDATE table_name
SET column1 = value1, column2 = value2, ...
[WHERE condition]
[RETURNING * | columns];  -- PostgreSQL
```

**Примеры**:  
— Обновление по условию:  
```sql
UPDATE users SET email = 'new@example.com' WHERE id = 5;
```
— Без `WHERE` — обновляются **все строки** (опасно!).  
— Использование `RETURNING` для аудита:  
```sql
UPDATE products 
SET price = price * 1.1 
WHERE category = 'electronics'
RETURNING name, price;
```

Оба оператора поддерживают транзакционность и могут быть откатаны при ошибке.
