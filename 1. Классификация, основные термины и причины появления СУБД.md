**Билет 1. Классификация, основные термины и причины появления СУБД**

### Основные термины
- **База данных (БД)** — упорядоченный набор структурированных данных.
- **СУБД** — система управления базами данных: ПО для создания, управления и взаимодействия с БД.
- **Таблица, запись, атрибут** — соответственно: отношение, кортеж и поле в реляционной модели.

### Классификация БД
- **По модели данных**: реляционные (PostgreSQL, MySQL), документные (MongoDB), ключ-значение (Redis), графовые (Neo4j).
- **По размещению**: централизованные vs распределённые.
- **По типу использования**: OLTP (оперативная обработка) vs OLAP (аналитика).

### Файловые системы vs СУБД
Файловые системы хранят данные как неструктурированные файлы — отсутствует целостность, безопасность, эффективный поиск и конкурентный доступ.  
СУБД появились для решения этих проблем:  
- обеспечение **целостности** (ограничения, транзакции),  
- **многопользовательский доступ** с управлением конфликтами,  
- **декларативный язык запросов** (SQL),  
- **восстановление после сбоев**.

Пример в PostgreSQL:
```sql
CREATE TABLE users (id SERIAL PRIMARY KEY, name TEXT NOT NULL);
```
— демонстрирует декларативное определение структуры с гарантией уникальности и типа.
