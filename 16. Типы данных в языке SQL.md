**Билет 16. Типы данных в языке SQL**

### Текстовые типы
- `CHAR(n)` — фиксированная длина (дополняется пробелами).
- `VARCHAR(n)` / `TEXT` — переменная длина; в PostgreSQL `TEXT` без ограничения.
- Пример:  
  ```sql
  CREATE TABLE users (login VARCHAR(30), bio TEXT);
  ```

### Числовые типы
- **Точные**:  
  - `INTEGER`, `BIGINT` — целые.  
  - `NUMERIC(p, s)` / `DECIMAL` — для финансов (точное представление).
- **Приблизительные**:  
  - `REAL`, `DOUBLE PRECISION` — с плавающей точкой (быстро, но неточно).

### Дата и время
- `DATE` — только дата (`2025-01-17`).  
- `TIME` — только время.  
- `TIMESTAMP` — дата + время.  
- `TIMESTAMPTZ` — с часовым поясом (рекомендуется в PostgreSQL).  
- Пример:  
  ```sql
  CREATE TABLE events (id SERIAL, at TIMESTAMPTZ DEFAULT NOW());
  ```

### Большие объекты
- `BYTEA` (PostgreSQL) или `BLOB` — бинарные данные (изображения, файлы).  
- `TEXT` или `CLOB` — большие тексты (в PG `TEXT` неограничен).

### JSON
- `JSON` — хранит как текст (проверка валидности при вставке).  
- `JSONB` (PostgreSQL) — бинарное представление, поддерживает индексы и эффективные запросы:  
  ```sql
  SELECT data->>name FROM logs WHERE data @> {"status": "error"};
  ```

Выбор типа влияет на производительность, объём и корректность операций.
