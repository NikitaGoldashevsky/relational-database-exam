**Билет 24. Операции переименования атрибутов и деления в реляционной алгебре**

### Переименование (ρ — rho)
- **Определение**: изменение имени отношения или его атрибутов без изменения данных.
- **Назначение**:  
  - Разрешение конфликтов имён при соединениях.  
  - Улучшение читаемости выражений.
- **Синтаксис**:  
  - ρ<sub>новое_имя</sub>(R) — переименовать отношение.  
  - ρ<sub>(a→x, b→y)</sub>(R) — переименовать атрибуты.
- **Пример**:  
  ```sql
  SELECT e.name AS employee, d.name AS department
  FROM employees e JOIN departments d ON e.dept_id = d.id;
  ```
  Соответствует: ρ<sub>(name→employee)</sub>(employees) ⨝ ρ<sub>(name→department)</sub>(departments)

### Деление (÷)
- **Определение**: находит кортежи из отношения R, которые связаны **со всеми** кортежами из отношения S.
- **Назначение**: запросы вида «найти всех студентов, записавшихся на **все** курсы».
- **Формально**:  
  R(A, B) ÷ S(B) = { a | ∀b ∈ S, (a, b) ∈ R }
- **Пример**:  
  Отношения:  
  - `Записи(студент, курс)`  
  - `Все_курсы(курс)`  
  Запрос:  
  ```sql
  SELECT студент FROM Записи
  GROUP BY студент
  HAVING COUNT(курс) = (SELECT COUNT(*) FROM Все_курсы);
  ```
  Это эмуляция деления в SQL.

### Свойства
- Деление **не коммутативно** и **не ассоциативно**.  
- Переименование — идемпотентная и обратимая операция.

Эти операции завершают выразительную мощность реляционной алгебры.
