**Билет 17. Оператор языка SQL для создания таблиц**

### Стандартный синтаксис `CREATE TABLE`
```sql
CREATE TABLE [IF NOT EXISTS] table_name (
  column_name data_type [column_constraint],
  ...
  [table_constraint]
);
```

### Ограничения (constraints)
- **Столбцовые**:  
  - `NOT NULL`, `UNIQUE`, `PRIMARY KEY`, `CHECK (condition)`, `DEFAULT value`.
- **Табличные**:  
  - `PRIMARY KEY (col, ...)`,  
  - `FOREIGN KEY (col) REFERENCES other(col)`,  
  - `UNIQUE (col, ...)`.

### Особенности PostgreSQL
- Поддержка `SERIAL` / `BIGSERIAL` (автоинкремент через SEQUENCE).  
- Расширенные типы: `JSONB`, `UUID`, `ARRAY`.  
- Условное создание: `CREATE TABLE IF NOT EXISTS`.  
- Наследование таблиц: `CREATE TABLE child () INHERITS (parent);`.

### Пример
```sql
CREATE TABLE products (
  id SERIAL PRIMARY KEY,
  name TEXT NOT NULL CHECK (length(name) > 0),
  price NUMERIC(10,2) DEFAULT 0.00 CHECK (price >= 0),
  tags TEXT[],
  created_at TIMESTAMPTZ DEFAULT NOW(),
  category_id INT REFERENCES categories(id) ON DELETE SET NULL
);
```

Этот пример демонстрирует типичную практику проектирования: целостность, безопасные значения по умолчанию и ссылки.
