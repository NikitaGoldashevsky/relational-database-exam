**Билет 21. Булевы операции над отношениями в реляционной алгебре**

### Общее требование
Все операции применяются к **совместимым по объединению** отношениям: одинаковое число атрибутов и совпадающие домены.

---

### Объединение (`∪`)
- **Определение**: все кортежи, принадлежащие хотя бы одному из отношений (без дубликатов).
- **Свойства**: коммутативна, ассоциативна.
- **SQL**: `UNION`  
  ```sql
  SELECT name FROM students
  UNION
  SELECT name FROM teachers;
  ```

### Пересечение (`∩`)
- **Определение**: кортежи, присутствующие в обоих отношениях.
- **Свойства**: коммутативна, ассоциативна.
- **SQL**: `INTERSECT`  
  ```sql
  SELECT product_id FROM orders_2023
  INTERSECT
  SELECT product_id FROM orders_2024;
  ```

### Разность (`−`)
- **Определение**: кортежи из первого отношения, отсутствующие во втором.
- **Свойства**: не коммутативна.
- **SQL**: `EXCEPT` (в PostgreSQL)  
  ```sql
  SELECT user_id FROM all_users
  EXCEPT
  SELECT user_id FROM banned_users;
  ```

### Декартово произведение (`×`)
- **Определение**: все возможные пары кортежей из двух отношений.
- **Результат**: отношение с количеством атрибутов = сумма атрибутов, строк = |R| × |S|.
- **SQL**: `CROSS JOIN`  
  ```sql
  SELECT * FROM colors CROSS JOIN sizes;
  ```
- **Примечание**: редко используется напрямую; чаще как основа для `JOIN`.

Эти операции — фундамент реляционной алгебры и оптимизации запросов.
