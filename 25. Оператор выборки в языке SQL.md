**Билет 25. Оператор выборки в языке SQL**

### Основные разделы `SELECT`
```sql
SELECT [DISTINCT] column_list
FROM table_sources
[WHERE condition]
[GROUP BY group_columns]
[HAVING group_condition]
[ORDER BY sort_columns]
[LIMIT n];
```

### Назначение разделов
- **`SELECT`**: перечисляет столбцы или выражения для вывода.  
  `DISTINCT` устраняет дубликаты.
- **`FROM`**: указывает источник данных (таблицы, подзапросы, CTE).
- **`WHERE`**: фильтрация строк **до** группировки.
- **`GROUP BY`**: агрегация по значениям (требуется при использовании `SUM`, `COUNT` и т.д.).
- **`HAVING`**: фильтрация групп **после** агрегации.
- **`ORDER BY`**: сортировка результата.
- **`LIMIT`** (PostgreSQL): ограничение числа строк.

### Пример
Найти 3 самых дорогих товара в категории «Электроника», с ценой > 1000:
```sql
SELECT name, price
FROM products
WHERE category = 'Электроника' AND price > 1000
ORDER BY price DESC
LIMIT 3;
```

Другой пример — агрегация:
```sql
SELECT department, AVG(salary) AS avg_salary
FROM employees
GROUP BY department
HAVING AVG(salary) > 50000
ORDER BY avg_salary DESC;
```

Порядок выполнения: `FROM` → `WHERE` → `GROUP BY` → `HAVING` → `SELECT` → `ORDER BY` → `LIMIT`.
